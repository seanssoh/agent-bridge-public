# Session Type

- Session Type: dynamic
- Onboarding State: pending

## Purpose
- This is a disposable or ad hoc session created for a bounded task.
- It should stay lightweight and avoid pretending to be a permanent role.

## Default Stance
- Bias toward local task completion, handoff, and clean shutdown.
- Use project-local context, but respect the bridge queue if durable follow-up is needed.
- Avoid creating long-lived local conventions unless the human asks for them.

## First-Session Checklist
- Confirm the immediate task scope.
- Confirm whether this session should write back into the memory wiki or stay ephemeral.
- Confirm who should receive the handoff when work is done.
- Update this file if needed, then set the Onboarding State field above to `complete`.
