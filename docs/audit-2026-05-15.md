# Audit archive — 2026-05-15

**Companion to**: [`docs/stabilization-plan-2026-05-15.md`](./stabilization-plan-2026-05-15.md)

This file captures the raw findings from the 5-track parallel audit dispatched on 2026-05-15 after v0.13.10 ship.

**Schema**: Each finding has a stable universal anchor across all 5 audits — `id`, `severity` (P0/P1/P2), `file:line` (or scope/category for non-line-anchored findings), `owner-stage` (plan stage), `status` (open/closed). Per-audit tables add domain-specific columns:
- **Audit A** (bug-surface): `+ category + one-line + risk`
- **Audit B** (stability): `+ category + one-line + risk`
- **Audit C** (isolation-v2): `+ bucket + site + current-linux + current-macos + proposed-gate`
- **Audit D** (refactor): `+ category + scope + effort`
- **Audit E** (test/docs): `+ category + one-line`

The universal anchor is what next-session iterates over; the extra per-audit columns are domain context.

**Next-session contract**: this file is the ground truth for stabilization work. New issues land at the bottom with the next sequential id; resolved items get `status: closed` with PR ref.

## Severity legend

- **P0** — actively blocking, fix before any other stage
- **P1** — should-fix, will surface under load / specific scenarios
- **P2** — nice-to-have, technical debt, document or defer

## Owner-stage legend

Per plan §"Stage plan" — `S1`, `S2`, `S1.5`, `S5-A` (Track A of S5), etc. `defer` = post-S10 / out-of-cycle.
